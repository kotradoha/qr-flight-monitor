카타르 ↔ 한국 항공편 모니터 · 운영 안내

KOTRA 도하무역관 운영용 한 장 요약 · 공개 페이지: kotradoha.github.io/qatar-korea-flight-monitor

① 이 페이지는 무엇인가요

카타르항공 한국 노선(QR858·QR859·QR862)의 운항 상태와 카타르 영공 폐쇄 여부를 15분마다 자동으로 불러와 교민에게 보여주는 대시보드입니다. 한국어·영어·아랍어를 지원합니다. 평상시에는 무역관이 손댈 필요가 전혀 없고, 아래 ③④는 실제로 상황이 생겼을 때만 쓰는 기능입니다.

② 화면 읽는 법

위쪽 박스 — 카타르 영공 폐쇄 여부 (신호등처럼 색으로 표시)

🟢 초록 · 폐쇄 공지 없음(정상) 🟠 주황 · 국제 회피 권고 발효(아직 안 막힘) 🔴 빨강 · 공식 폐쇄 확인 ⚪ 회색 · 일시적으로 확인 중

그 아래 — 항공 운항 특기사항: 특정 편의 결항·지연이 실제로 생겼을 때만 별도로 뜹니다. 영공 색깔과는 무관합니다. 즉 하늘길은 정상(초록)이어도 어떤 편만 결항하면, 위는 초록 그대로이고 이 칸에만 그 결항이 표시됩니다.

※ 표에는 현재 비행 중이거나 앞으로 약 1주일간 예정된 편만 나오며, 예정 시각은 카타르항공 공식 스케줄과 같습니다.

③ 카타르 영공 “폐쇄(빨강)”을 직접 켜고 끄기 가장 중요

빨강은 결항이 아무리 많아도 저절로 켜지지 않습니다. 카타르 정부·항공당국·공신력 있는 뉴스가 공식적으로 폐쇄를 발표한 것을 확인하시면, 아래 방법으로 직접 빨강을 켜 주세요. (자동 감지도 있지만, 가장 확실한 것은 이 수동 스위치입니다.)

방법 (브라우저에서 클릭만 하면 됩니다)

  1. 저장소 열기 → github.com/kotradoha/qatar-korea-flight-monitor 의 docs 폴더 → manual_notice.json 클릭
  2. 오른쪽 위 연필(✏️ Edit) 아이콘 클릭
  3. 아래 두 줄을 목적에 맞게 고칩니다 (아래 예시 참고)
  4. 오른쪽 위 초록색 Commit changes → 다시 Commit changes 클릭
  5. 1~2분 뒤 공개 페이지에 자동 반영됩니다

폐쇄 켜기 (빨강으로)

"airspace_status": "closed", "airspace_status_source": "공식 공지 링크"

출처 링크(뉴스·공항·정부 공지 주소)를 꼭 함께 넣어 주세요.

정상으로 되돌리기 (평상시)

"airspace_status": null, "airspace_status_source": ""

null 은 따옴표 없이 그대로 씁니다. 평소엔 항상 이 상태로 둡니다.

※ 만약 자동 감지가 실수로 빨강을 띄웠다면(실제로는 정상인데), "airspace_status"를 "open" 으로 두면 강제로 초록(정상) 처리됩니다. 상황이 끝나면 다시 null 로 되돌려 주세요.

④ 긴급 공지 한 줄 띄우기 (선택)

“7/20 도하–서울편 결항 안내” 같은 안내 문구를 표 위에 띄우고 싶을 때는 같은 manual_notice.json 파일의 items 에 한 줄 추가합니다. 비워두면 아무것도 안 뜹니다.

"items": [ { "title": "[카타르항공] 도하–서울 7/20 결항 안내", "url": "https://www.qatarairways.com/en/travel-alerts.html" } ]

문구가 끝나면 "items": [] (빈 대괄호)로 되돌리면 사라집니다.

⑤ 정기편이 한동안 안 뜰 때 (임시 미운영)

카타르항공이 특정 정기편을 한동안 운휴(예: 주간편 QR862·QR863 일시 중단)하면, 표를 지우지 않고 그 카드에 ⏸ 임시 미운영으로 표시됩니다. 매일편은 근접 며칠간 편이 계속 안 잡히면 자동으로 잡히지만, 주간편이나 장기 운휴는 아래처럼 직접 지정하는 것이 확실합니다. 같은 manual_notice.json 의 suspended_flights 를 고칩니다.

"suspended_flights": { "QR863": { "until": "2026-08-31", "note": "임시 운휴 안내", "note_en": "Temporarily suspended" } }

until(재개 예정일)·note(안내 문구)는 생략 가능합니다. 간단히 "suspended_flights": ["QR863"] 처럼 편명만 적어도 됩니다. 운항이 재개되면 "suspended_flights": {} (빈 중괄호)로 되돌려 주세요.

⑥ 갱신이 멈춘 것 같을 때

페이지 상단의 “마지막 확인” 시각이 한참 전(수십 분 이상)에서 안 바뀌면, 저장소의 Actions 탭을 열어 Update flight data 워크플로가 초록색으로 돌고 있는지 확인하세요. 빨간 X(실패)가 보이면 클릭해 내용을 확인하시고, 필요하면 저에게 알려 주시면 점검해 드리겠습니다.
